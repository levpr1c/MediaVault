# MediaVault

[![EN](https://img.shields.io/badge/lang-EN-blue.svg)](README.md)

**MediaVault** — это программа для твоего компьютера, которая собирает все картинки и видео в одну удобную галерею,
автоматически находит к ним теги (ключевые слова) на booru-сайтах, и показывает всё в красивом интерфейсе.

> 🎯 **Для кого:** Для тех, у кого много артов, фотографий или видео, и кто хочет быстро находить нужное по тегам.
>
> 🔒 **Всё локально:** Никаких облаков, все данные на твоём компьютере.

---

## 🚀 Быстрый старт

### AUR (Arch Linux)

```bash
yay -S mediavault-bin
# или
paru -S mediavault-bin
mediavault
# → http://localhost:5050
```

FFmpeg и GNOME Keyring устанавливаются автоматически зависимостями пакета.

### Вручную (любой дистрибутив)

```bash
git clone https://github.com/levpr1c/MediaVault
cd MediaVault
python3 -m venv venv
venv/bin/pip install -r requirements.txt
venv/bin/python src/web_app.py
# → http://localhost:5050
```

> FFmpeg нужен для превью видео. Установи: `sudo apt install ffmpeg` (Linux) или `brew install ffmpeg` (macOS).
> `python3-keyring` — опционально, для хранения API-ключей в системной связке. Если пакет не установлен, ключи хранятся в settings.json.

## ✨ Возможности

| Возможность | Что это даёт |
|-------------|--------------|
| **🔍 Tagfetch** | Ищет теги на Rule34 и Danbooru по MD5-хешу файла. Два режима: Manual (по одному) и Auto (все сразу, с прогрессом) |
| **🖼️ Галерея** | Masonry/Grid/List раскладки, поиск по имени и тегам, фильтрация |
| **💡 Лайтбокс** | Просмотр на весь экран с зумом, навигацией, редактированием тегов |
| **📚 Комиксы** | Собирай подборки, читай в Scroll (лента) или Lightbox (страницы) режиме |
| **🏷️ Теги** | Категории с цветами (artist/character/copyright/general), авто-теги (photo/video/animated/sound), теги соотношения сторон |
| **👥 Пользователи** | Админ (всё можно) и обычный пользователь (только просмотр) |
| **🌙 Тема** | Тёмная и светлая, переключается без перезагрузки |
| **🌐 Язык** | Русский и английский, переключается на лету |
| **🎨 Three.js фон** | Анимированный шейдерный фон на главной и странице логина, работает на GPU |

## 🧭 Структура приложения

MediaVault состоит из трёх частей:

```
┌─ MediaVault (просмотр) ─────────────────────┐
│  Галерея, комиксы, лайтбокс                 │
│  Доступно всем (admin + user)               │
└──────────────────────────────────────────────┘
┌─ Content Management (редактирование) ───────┐
│  Теги, категории, комиксы, файлы            │
│  Только для admin                           │
└──────────────────────────────────────────────┘
┌─ Admin / Settings (настройки) ──────────────┐
│  Пользователи, API-ключи, БД, тема, язык    │
│  Только для admin                           │
└──────────────────────────────────────────────┘
```

### Технологии

| Слой | Технология |
|------|-----------|
| Сервер | Python / Flask (один файл, ~7700 строк) |

| Обслуживание | 13 | БД, миниатюры, очистка, дедупликация |

## 🏗 Сборка бинарника

```bash
# Установить PyInstaller (один раз)
venv/bin/pip install pyinstaller

# Собрать
venv/bin/pyinstaller mediavault.spec --clean --noconfirm

# Запустить
./dist/mediavault/mediavault
```

FFmpeg **не встроен** в бинарник — установи в систему: `sudo apt install ffmpeg`.
Пути к данным те же, что и при запуске из исходников (`~/.config/MediaVault/`, `~/.local/share/MediaVault/`).

## Arch Linux (AUR)

Установка из AUR (`mediavault-bin`):

```bash
yay -S mediavault-bin
# или
paru -S mediavault-bin
```

Пакет зависит от `ffmpeg` и опционально от `gnome-keyring` для шифрования API ключей.

### Релизный процесс

```bash
# 1. Собрать бинарник
venv/bin/pyinstaller mediavault.spec --clean --noconfirm

# 2. Переименовать для релиза
mv dist/mediavault dist/mediavault-linux-amd64

# 3. Посчитать контрольную сумму
sha256sum dist/mediavault-linux-amd64

# 4. Создать GitHub Release v1.0.0, прикрепить dist/mediavault-linux-amd64
# 5. Обновить PKGBUILD в packaging/aur/mediavault-bin/ (sha256sum, pkgver)
# 6. Запушить в AUR
```

## 📁 Где что лежит

| Данные | Путь |
|--------|------|
| База данных (теги, кеш, комиксы) | `~/.local/share/MediaVault/MediaVaultDataBase.db` |
| Настройки (тема, media_dir, ключи) | `~/.config/MediaVault/settings.json` |
| Лог отладки (при `--debug`) | `~/.local/share/MediaVault/debug.log` |
