# Часто задаваемые вопросы (FAQ)

> Ответы на вопросы, которые чаще всего возникают у новых пользователей MediaVault.

---

## Установка и запуск

### ❓ Что нужно установить, чтобы запустить MediaVault?
**Минимум:** Python 3, библиотеки из `requirements.txt`.
**Для видео:** дополнительно FFmpeg (нужен для превью видео и определения звука).
**Для хранения API-ключей:** опционально `keyring` (GNOME Keyring; KDE KWallet в будущем).

```bash
# Создать виртуальное окружение (один раз)
python3 -m venv venv

# Установить зависимости
venv/bin/pip install -r requirements.txt

# Запустить
venv/bin/python src/web_app.py
```

### ❓ Почему при запуске пишет «Address already in use»?
Порт 5050 уже занят другой программой. Либо останови её, либо запусти на другом порту:

```bash
venv/bin/python src/web_app.py --port 8080
```

### ❓ Как включить режим отладки?
```bash
venv/bin/python src/web_app.py --debug
```
Включит:
- Авто-перезагрузку при изменении кода
- Подробные цветные логи в консоли
- Файл `~/.local/share/MediaVault/debug.log`

### ❓ Как сменить порт или интерфейс?
```bash
venv/bin/python src/web_app.py --port 9090 --bind 192.168.1.100
```

## Первый запуск

### ❓ Захожу на `http://localhost:5050` — вижу пустую страницу. Что делать?
1. Настрой медиа-папку (Settings → Appearance → Media Folder)
2. Нажми **Scan Folder** — MediaVault просканирует папку и добавит файлы
3. Либо используй **Tagfetch** для поиска тегов по MD5

Подробнее: [Руководство пользователя → Первый запуск](docs/user-guide.md#первый-запуск-настройка-медиа-папки)

### ❓ Где хранятся файлы и настройки?
Всё локально, на твоём компьютере:

| Что | Где лежит |
|-----|-----------|
| База данных (теги, кеш, комиксы) | `~/.local/share/MediaVault/MediaVaultDataBase.db` |
| Настройки (тема, media_dir, API-ключи) | `~/.config/MediaVault/settings.json` |
| Лог отладки (при --debug) | `~/.local/share/MediaVault/debug.log` |

### ❓ Нужны ли API-ключи для Rule34 и Danbooru?
- **Rule34** — нужен UID и Key (можно получить на Rule34.xxx после регистрации)
- **Danbooru** — нужны Login и API Key (в настройках профиля на Danbooru)
- Если ключей нет — MediaVault будет работать, но не сможет искать теги на этих сайтах. Авто-теги (photo/video/animated/sound) и просмотр галереи будут работать в любом случае.

## Использование

### ❓ Почему теги не находятся?
1. Проверь, что API-ключи введены и сохранены (Settings → API Keys)
2. Файл должен быть в медиа-папке
3. Файл должен быть известен booru-сайту (существовать в их базе)
4. Попробуй **Auto Scan** — он переберёт все файлы

### ❓ В чём разница между Manual и Auto в Tagfetch?
- **Manual** — выбери один файл, нажми «Fetch Tags». Результаты видны сразу, можно сохранить или отклонить.
- **Auto** — сканирует все файлы в папке по очереди с живым прогрессом (SSE). После сканирования можно сохранить всё сразу.

### ❓ Что значат цвета тегов?
Категории Danbooru с цветами:

| Цвет | Категория | Пример |
|------|-----------|--------|
| Голубой | Artist (художник) | `kagami_hiroshi` |
| Оранжевый | Character (персонаж) | `hatsune_miku` |
| Фиолетовый | Copyright (серия) | `vocaloid` |
| Зелёный | General (общее) | `cat`, `1girl` |
| Синий | Meta (служебное) | `highres`, `commentary` |

### ❓ Как создать комикс?
1. Перейди в **Content Management → COMICS EDITOR** (только для админа)
2. Нажми «Create Comic»
3. Назови комикс
4. Выбери файлы из галереи (можно искать и сортировать)
5. Перетащи страницы в нужном порядке
6. Сохрани

Или через MediaVault Comics: открой файл в просмотрщике, нажми «Create comic».

### ❓ Как удалить файл из базы?
Удали его из медиа-папки и пересканируй. Или используй **Content Management → Files** (найди файл, открой карточку, удали).

### ❓ Что такое «авто-теги»?
Это теги, которые MediaVault присваивает сама без обращения к booru-сайтам:
- `photo` — обычное изображение
- `video` — видеофайл
- `animated` + `gif` — гифка
- `sound` — есть аудиодорожка
- `16:9`, `4:3` — соотношение сторон
- `photo` — для изображений

## Проблемы

### ❓ Не грузятся миниатюры (превью)
```bash
# Перегенерировать все миниатюры
curl -X POST http://localhost:5050/api/regenerate_thumbnails

# Или через Settings → Database → Regenerate All Thumbnails
```

### ❓ Медленно работает
Галерея тормозит при большом количестве файлов? Попробуй:
- Уменьшить размер страницы (иконка Pg: в сайдбаре)
- Переключиться на Grid или List layout (но не Masonry)
- Включить `--debug` и посмотреть, что в логах

### ❓ Сбросить всё и начать заново
**Settings → Database → Clean All** — очистит кэш тегов, миниатюры, базу данных. После этого будет автоматическое пересканирование.

### ❓ Комиксы не сохраняются
Убедись, что ты вошёл как **admin**. Только администратор может создавать и редактировать комиксы. Если ты вошёл как `user` — кнопки редактирования не появятся.

### ❓ Не работает смена языка
Кнопка с глобусом в правом верхнем углу хедера. Язык меняется на лету, без перезагрузки. Если не сработало — обнови страницу вручную (`F5`).

### ❓ Что такое «бэкенды» в админ-панели?
Бэкенды определяют, какой модуль используется для получения тегов с каждого сайта:
- **API Raw** — для Rule34 и Danbooru (прямые API)
- **API Raw** — для Rule34, Danbooru и NHentai (прямые API)
- **Gallery-DL** — для Kemono и Coomer (через gallery-dl CLI), а также для NHentai (по умолчанию)
Настройка: Admin Panel → Backends. Если бэкенд не выбран — используется `api_raw` по умолчанию.

### ❓ Зачем нужны иконки сайтов?
Иконки Rule34, Danbooru, NHentai, Kemono, Coomer используются в интерфейсе админ-панели (секция Backends) для визуальной идентификации сайтов. Иконки — реальные favicon (скачанные с сайтов), доступны через `window.SiteIcons.getIconImg('rule34', 16)`.

### ❓ Что такое Folder Settings в админке?
Раздел Folders в Admin Panel позволяет настроить имена подпапок в вашей медиа-директории:
- **Gallery folder** (по умолч. `Gallery`) — файлы из этой подпапки считаются галереей
- **Comics folder** (по умолч. `Comics`) — файлы из этой подпапки считаются комиксами
Это используется для фильтрации в `/api/gallery?folder=gallery|comics`.

### ❓ Зачем нужен Browser Cache в настройках?
Определяет, как долго браузер кэширует медиа-файлы и превью:
- **Default** — кэш на сутки (`max-age=86400`)
- **Reduced** — кэш на час (`max-age=3600`)
- **No Cache** — без кэширования (`no-cache`)
Полезно при разработке или частом обновлении файлов. Настройка: Settings → Appearance → Browser Cache.

### ❓ Как работать с Kemono/Coomer?
Используется встроенная интеграция через `gallery-dl`:
1. Убедись что `gallery-dl` установлен в системе
2. Открой `/kemono-import` или используй API: `POST /api/kemono/download` с URL поста
3. Файлы скачиваются в `media_dir/Downloads/kemono/`
Поддерживаются зеркала: `.su`, `.cr`, `.cv`, `.party`, `.so`, `.us`, `.co` для kemono и coomer.

### ❓ Как искать теги на NHentai?
Через страницу поиска `/nhentai-search` (кнопка «Comics Fetch» на главной) или через API: `GET /api/nhentai/search?q=query`. Использует `GalleryDlBackend` (gallery-dl) для поиска и получения данных.

## Безопасность

### ❓ Насколько безопасно хранить ключи?
- По умолчанию MediaVault использует **GNOME Keyring** (системную связку паролей)
- Можно переключиться на хранение в **settings.json** (обычный текст) — удобно, но менее безопасно
- Настройка: Settings → Appearance → Credential Backend

### ❓ Как добавить нового пользователя?
Только администратор:
1. Admin Panel → Users → Add User
2. Введи имя и пароль
3. Выбери роль (`admin` или `user`)

### ❓ Я забыл пароль администратора!
```bash
# Сбросить пароль первого админа через SQLite
sqlite3 ~/.local/share/MediaVault/MediaVaultDataBase.db "UPDATE users SET password='' WHERE username='your_username'"
# После этого зайди без пароля и установи новый
```

## Разное

### ❓ На домашней странице анимированный фон — это тормозит?
Это Three.js шейдерный фон. Работает на GPU, не нагружает процессор. Можно отключить в Settings → Appearance → "Three.js background".

### ❓ Как создать комикс из MediaVault?
Если ты admin, в конце сетки комиксов есть карточка с "+". Нажми её, выбери название и файлы в ComicsPicker, сохрани.

### ❓ Почему настройки не сохраняются?
Проверь права на запись в `~/.config/MediaVault/`. Если проблема сохраняется — удали `.tmp` файлы в этой папке.
