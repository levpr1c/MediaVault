# MediaVault — Полное руководство пользователя

## Что это и зачем

**MediaVault** — это локальный веб-инструмент, который:

1. **Собирает все твои медиафайлы** в одну галерею (не надо раскладывать по папкам)
2. **Находит теги** с Rule34.xxx и Danbooru.donmai.us по MD5-хешу файла
3. **Тегирует коллекцию** — группирует теги по категориям с цветами
4. **Показывает** всё в удобной галерее с Masonry/Grid/List раскладкой
5. **Даёт лайтбокс** с зумом, навигацией, редактированием тегов
6. **Поддерживает комиксы/мангу** — собирай подборки и читай в ридере
7. **Работает в 2 темах** (светлая/тёмная) и **2 языках** (русский/English)
8. **Видео и аудио** — превью, стриминг, Range-запросы

Всё работает локально — никаких облаков, всё на твоём компьютере.

## Быстрый старт

```bash
# Установка зависимостей (один раз)
python3 -m venv venv
venv/bin/pip install -r requirements.txt

# Запуск
venv/bin/python src/web_app.py
# → http://localhost:5050
```

> `keyring` — опционально, для хранения API-ключей в системной связке (GNOME Keyring). В будущем — поддержка KDE KWallet. Если пакет не установлен, ключи хранятся в settings.json.

Открой браузер. Сразу увидишь домашнюю страницу с тремя блоками:
- **MEDIAVAULT** — галерея и комиксы (просмотр)
- **CONTENT MANAGEMENT** — управление тегами, комиксами, TagFetch (только для admin)
- **ADMIN & SETTINGS** — настройки и панель управления

## Первый запуск (настройка медиа-папки)

1. Запусти сервер
2. Перейди в **Settings** (иконка шестерёнки в правом верхнем углу или `/settings`)
3. Открой вкладку **Appearance**
4. Укажи **Media directory** — абсолютный путь к папке с артами/фотками/видео
5. Нажми **Save**
6. Нажми **Scan** — сервер просканирует папку и добавит файлы в базу

### Учётная запись администратора

При первом запуске автоматически создаётся учётная запись с ролью `admin`:
- **Логин:** `admin`
- **Пароль:** `admin`

Просто открой `/login` и войди. После входа доступны **Content Management** и **Admin Panel**.

**Важно:** смени пароль в настройках (`/settings` → вкладка **Account**) сразу после первого входа.

### API-ключи (опционально)

Для поиска тегов на Rule34.xxx и Danbooru нужны API-ключи:
1. Открой **Settings** → вкладка **API Keys**
2. Введи:
   - **Rule34 UID / Key** — для Rule34.xxx
   - **Danbooru Login / Key** — для Danbooru
3. Нажми **Save**
4. Если ключи не ввести — поиск тегов на этих сайтах работать не будет

### Как работает автосканирование

- При запуске сервер проверяет количество файлов в медиа-папке
- Если файлов стало больше — добавляет новые в базу
- Если ничего не изменилось — пропускает
- Добавляет auto-теги (`photo`, `video,animated`, `sound` для аудио, `sound` для видео со звуком) — без API
- Настоящие теги с Rule34/Danbooru — через Tagfetch
- Скрытые папки (`.thumb_cache`, `.Trash-1000`) и скрытые файлы игнорируются

## Settings — Настройки

**Маршрут:** `/settings` (доступен всем авторизованным пользователям)

Страница настроек с четырьмя вкладками (табами) в хедере:
- Вкладки **API Keys** и **Database** отображаются только для пользователей с ролью `admin`
- Вкладки **Appearance** и **Account** доступны всем авторизованным

### Вкладка Appearance (Внешний вид)

| Раздел | Что можно настроить |
|--------|---------------------|
| **Тема** | Переключение светлой/тёмной темы (те же кнопки ☀️/🌙, что в хедере) |
| **Медиа-папка** | Указать путь к папке с файлами, кнопки **Save** (сохранить) и **Scan** (сканировать) |
| **Визуальные эффекты** | Включить/выключить размытие, анимации и переходы (toggle) |
| **Three.js Background** | Включить/выключить анимированный фон на домашней странице и странице входа (toggle). По умолчанию включён |

Визуальные эффекты управляют CSS-анимациями, размытием фона модалок и переходами. Отключение может ускорить работу на слабых устройствах.

**Three.js Background** — анимированный частичный фон (particle field) на страницах `/` и `/login`. Работает через Three.js (самостоятельно загружается с сервера, без CDN). Если фон отключён — эти страницы показываются на чистом фоне (темы). Отключается независимо от визуальных эффектов.

### Вкладка API Keys (API-ключи)

Два блока:

**API Credentials:**
- Rule34 UID / Key
- Danbooru Login / Key
- Кнопка **Save**

**Credential Backend** — выбор хранилища для API-ключей:

| Бэкенд | Где хранятся ключи | Безопасность |
|--------|-------------------|--------------|
| **Keyring** | GNOME Keyring (libsecret) | Высокая — зашифровано системой |
| **Plain Text** | `settings.json` в открытом виде | Низкая — видно в файле |

Доступность бэкендов определяется системой: Keyring доступен если установлен пакет `python3-keyring` и есть GNOME Keyring.

### Вкладка Database (База данных)

Сетка инструментов для обслуживания БД:

| Инструмент | Описание | Опасный? |
|------------|----------|----------|
| **Export DB** | Скачать файл базы данных | Нет |
| **Import DB** | Загрузить `.db` файл (заменяет текущую БД) | Да |
| **Clean Duplicates** | Удаляет дубликаты записей | Нет |
| **Clean Thumbnail Cache** | Очищает кэш превью (BLOB в БД) | Да |
| **Clean Tag Cache** | Очищает in-memory кэш API-ответов | Да |
| **Clean Database** | Очищает таблицы `files`, `scan_results`, `auto_scan` | Да |
| **Regenerate All Thumbnails** | Пересоздаёт миниатюры для всех файлов | Нет |
| **Generate Missing** | Создаёт миниатюры только для файлов без превью | Нет |
| **Clean All** | Всё сразу + авто-рескан | Да |

**Regenerate All Thumbnails** использует SSE-поток — прогресс отображается в реальном времени:
- Зелёная полоса прогресса
- Счётчик: `текущий / всего (сгенерировано: N, пропущено: M, ошибок: K)`
- Кнопка **Cancel** — прервать генерацию
- После завершения — всплывающее уведомление

> **Atomic save:** при сохранении настроек используется атомарная запись в файл — данные не теряются даже при сбое в середине сохранения. Ранее существовала race condition при параллельном сохранении разных разделов (например, темы и API-ключей); теперь каждый вызов `save_settings()` полностью перезаписывает файл, исключая частичное обновление.

### Вкладка Account (Аккаунт)

Два блока:

**Смена имени пользователя:**
- Поле нового имени
- Поле текущего пароля (для подтверждения)
- Кнопка **Change Username**

**Смена пароля:**
- Текущий пароль
- Новый пароль
- Подтверждение нового пароля
- Кнопка **Change Password**

## Content Management — Управление контентом (только admin)

**Маршруты:** `/content-mgmt/*`

Блок на главной странице, доступный только администраторам.
Содержит три колонки:

### TAGFETCH — поиск тегов

#### Ручной режим (Manual)

**Маршрут:** `/content-mgmt/tags-manual`

Работа с одним файлом за раз:

1. **Файловый браузер (слева):**
   - Открывай папки кликом по папкам
   - Breadcrumb (хлебные крошки) — клик по любой части пути
   - Статус файла показывается иконкой:
     - 💾 — теги уже в БД
     - 🔍 — теги найдены (кеш)
     - 🚫 — теги не найдены
     - ❌ — нет тегов (только auto-tags)
   - Сортировка: по имени / сначала новые / сначала старые
   - Фильтрация: All / No tags / Found / Not found / In DB

2. **Выбор файла:** кликни по файлу — справа появится превью + информация

3. **Поиск тегов:** нажми **Fetch Tags** — сервер ищет по MD5 на Rule34 и Danbooru

4. **Сохранение:** после фетча — кнопка **Add from...** (Rule34, Danbooru, или оба)

5. **Fetch All:** обработать все файлы в папке, затем **Save All** — сохранить всё найденное разом

#### Автоматический режим (Auto)

**Маршрут:** `/content-mgmt/tags-auto`

Обрабатывает все файлы в медиа-папке через SSE-поток:

1. Нажми **Start Auto Scan**
2. Сервер проходит по файлам, для каждого проверяет статус и фетчит теги
3. Результаты — карточки с превью и найденными тегами
4. **Save All** после завершения — сохранить все найденные теги

**Статусы файлов:** `ok` (найдены), `skip` (уже сканирован), `no_tags` (нет на сайтах)

### TAGS — управление тегами

**Маршруты:**
- `/content-mgmt/tags-manage` — просмотр и управление тегами, drag-and-drop по категориям
- `/content-mgmt/tags-group` — управление категориями (группами) тегов с цветами

### COMICS EDITOR — редактор комиксов

**Маршрут:** `/content-mgmt/comics-edit`

Полноценный редактор комиксов:
- Создание новых комиксов через ComicsPicker модалку
- Drag-and-drop страниц
- Выбор обложки
- Удаление комиксов
- Просмотр в ридере

## MediaVault — галерея (только просмотр)

**Маршрут:** `/mediavault/gallery`

### Раскладки (Layout)

Три кнопки в тулбаре:

| Кнопка | Режим | Как выглядит |
|--------|-------|-------------|
| Квадратики 2×2 | **Masonry** | Как Pinterest — колонки фиксированной ширины, элементы разной высоты |
| Сетка | **Grid** | Ровная таблица — все ячейки одинакового размера |
| Полоски | **List** | Вертикальный список на всю ширину |

### Размер превью

S (160px) — мелко, M (220px) — средне, L (300px) — крупно.

### Количество на странице

- **All** — все сразу (без пагинации)
- **30 / 60 / 90** — количество файлов на странице

### Режим "Tagged"

Кнопка **Tagged** — показывает только файлы у которых есть теги из API (не auto-tags).

### Сортировка по дате

Кнопка ↕ в тулбаре:

1. **По имени** — стандартная сортировка (по алфавиту)
2. **Сначала новые** — от новых к старым
3. **Сначала старые** — от старых к новым

### Поиск по тегам

В сайдбаре слева:
- Введи тег — найдутся все файлы с этим тегом
- Можно несколько тегов через запятую
- `-тег` исключает файлы с этим тегом
- Переключение режима: **Name** / **Tag** / **Both**

### Пагинация

Снизу страницы (и сверху в тулбаре):
- Кнопки с номерами страниц
- «…» если страниц много
- Текущая страница выделена

### Теги в сайдбаре

Справа — блок **CATEGORIZED**:
- Теги сгруппированы по Danbooru-категориям (artist, character, copyright, general)
- Показывается топ-20 тегов на категорию
- Если тегов больше — ссылка `+N more`
- Счётчик показывает количество ВСЕХ файлов с этим тегом
- Категории имеют цвета: artist=красный, character=зелёный и т.д.
- Клик по тегу — поиск по этому тегу

### Bulk Tag (admin)

Кнопка **Bulk Tag** (видна только admin):
- Включает режим выбора файлов
- Клик по файлам — добавляет в выделение
- После выбора — добавить теги всем выделенным разом

### Управление категориями (admin)

Кнопка **Group Colors** в сайдбаре (только admin):
- Добавление/удаление категорий
- Смена цвета категории
- Привязка тега к категории (через модалку)

## Лайтбокс (Lightbox)

Открывается кликом по любому файлу в галерее.

### Режимы просмотра

| Режим | Как включить | Что делает |
|-------|-------------|------------|
| **Fit** | По умолчанию | Изображение вписано в окно |
| **Scroll (↕)** | Автоматически для длинных картинок (>2.5:1) | На всю ширину, скролл вниз |
| **Full (1:1)** | Клик по картинке | Натуральный размер, зум колёсиком + Ctrl, drag |

### Управление

| Действие | Результат |
|----------|-----------|
| ← / → (клавиши) | Предыдущий / следующий файл |
| Тап слева/справа (мобильные) | Предыдущий / следующий файл |
| Esc | Закрыть |
| Ctrl + колёсико мыши | Зум |
| Drag | Перетащить (в Scroll/Full режиме) |

### Редактор тегов в лайтбоксе

Справа панель с тегами:
- Поле ввода — начни печатать, автокомплит подскажет
- Теги показываются с категориями и цветами
- ✕ на чипе — удалить тег
- Enter — добавить тег
- Кнопка **Save** — сохранить изменения
- Кнопка **Edit Categories** — открыть управление категориями

### Видео в лайтбоксе

- Поддержка Range-запросов — можно перематывать, ставить на паузу
- Стриминг — видео грузится по частям, не ждёт полной загрузки

### Отдельная страница просмотра

Кнопка ↗ в тулбаре лайтбокса — открывает файл в отдельной вкладке:
- Компактный топбар (40px)
- Навигация: тап по левой/правой части (12% экрана)
- Теги: выезжают снизу, переключаются кнопкой
- Порядок навигации сохраняется из галереи

## Comics — комиксы и манга

### Просмотр (MediaVault)

**Маршрут:** `/mediavault/comics`

В хедере отображается «MEDIAVAULT» (как и на других страницах просмотра).

Сетка карточек:
- Каждая карточка — обложка + название
- Клик по карточке → открывается ридер
- Для admin: в конце сетки — карточка **«+»** (Add Comic), которая открывает ComicsPicker модалку:
  1. Введи название комикса
  2. Файловый браузер: поиск, сортировка, hover preview
  3. Comics Map: drag-and-drop страниц, выбор обложки
  4. **Save** — комикс создан
- Для обычных пользователей: если комиксов нет — показывается сообщение «Нет комиксов»

### Чтение комикса

**Маршрут:** `/mediavault/view?id=N`

**Scroll mode (по умолчанию):**
- Все страницы вертикально, как лента (вебтун/манхва)
- Скролль мышью, пальцем или колесом
- В правом нижнем углу плавающий счётчик — показывает текущую/всего
- **F** или кнопка — переключиться в lightbox

**Lightbox mode:**
- Одна страница на весь экран
- ← / → (клавиши или тап по краям) — листать
- Точки внизу — прогресс чтения
- **F** или кнопка — вернуться в scroll mode

## Роли пользователей

### Администратор (`admin`)

Полный доступ ко всем функциям:
- **Content Management** — TagFetch (Manual/Auto), управление тегами и категориями, редактор комиксов
- **MediaVault** — bulk tag, управление категориями, создание комиксов
- **Comics** — создание (+ карточка в сетке) и удаление комиксов
- **Settings** — все 4 вкладки: Appearance, API Keys, Database, Account
- **Admin Panel** (`/admin`) — управление пользователями, данными, API-ключами

### Обычный пользователь (`user`)

Только просмотр. Без кнопок редактирования:
- **MediaVault Gallery** — просмотр, лайтбокс, фильтрация по тегам
- **Comics** — чтение комиксов (без создания/удаления); если комиксов нет — видит сообщение «Нет комиксов»
- **Settings** — только вкладки Appearance и Account (API Keys и Database скрыты)
- **Admin Panel** — не отображается ни в выпадающем меню, ни в главном блоке

**Безопасность:** фронтенд скрывает кнопки, но бэкенд всегда проверяет права через `@admin_required` — 403 Forbidden для не-админов.

## Admin Panel — Панель управления

Отдельная страница, не входящая в Settings (доступна из выпадающего меню аккаунта на главной или напрямую `/admin`).
Не-админы не видят ссылку на Admin Panel ни в меню, ни на главной.

Панель с тремя разделами (иконки в хедере):

### Пользователи

Таблица пользователей:
- Создание/удаление пользователей
- Назначение ролей (`admin` / `user`)
- Смена пароля любому пользователю
- Защита: нельзя удалить последнего администратора

### База данных

Те же инструменты, что в Settings → Database, плюс:
- Прогресс-бар с SSE для регенерации превью
- Кнопка **Cancel** для прерывания

### API-ключи

Поля для Rule34 и Danbooru, выбор Credential Backend, медиа-папка — дублирует функционал Settings, но в отдельном интерфейсе (для администрирования).

## Тема оформления и язык

Переключение в хедере (или на главной странице):

- **☀️/🌙** — переключение между светлой и тёмной темой (SVG-иконки солнца/луны)
- **EN/RU** — переключение языка (SVG-иконка глобуса + текст), без перезагрузки страницы

При переключении темы иконка анимируется — поворот на 360° (0.5s).
Язык переключается мгновенно через JavaScript, обновляя все `data-i18n` атрибуты на странице.

Всё запоминается между сессиями:
- Тема — в `~/.config/MediaVault/settings.json`
- Язык — в `localStorage` + cookie `mediavault_lang`

## Хедер (верхняя панель)

Хедер адаптируется под текущую страницу:

| Элемент | Описание |
|---------|----------|
| **Название раздела** | Слева — бренд страницы (Settings, Admin Panel, MEDIAVAULT и т.д.) |
| **Табы навигации** | Вкладки для переключения между подразделами (зависит от страницы) |
| **Кнопка Home** | Справа — иконка домика, возврат на главную |
| **☀️/🌙** | Переключение темы (SVG-иконки солнца/луны) |
| **EN/RU** | Переключение языка (глобус + текст) |

На мобильных устройствах хедер адаптируется: часть элементов скрывается, появляется гамбургер-меню (drawer).
На странице комиксов (MV) в хедере отображается «MEDIAVAULT», а в табах — Gallery и Comics.

## Визуальные эффекты (Effects)

Настройка в **Settings** → **Appearance** или через `Shared.toggleEffects()`.

Когда эффекты **включены**:
- Размытие (backdrop-filter) фона модальных окон
- Анимации появления/исчезания элементов
- Плавные переходы при смене темы

Когда эффекты **выключены** (`data-no-effects` на `<html>`):
- Все анимации отключены
- Размытие фона отключено
- Полезно на слабых устройствах или для быстрой работы

### Three.js Background (анимированный фон)

Настройка в **Settings** → **Appearance** или через системный тумблер.

Когда фон **включён** (значение по умолчанию):
- На домашней странице (`/`) и странице входа (`/login`) показывается анимированное поле частиц (Three.js)
- Частицы переливаются цветом темы (акцентный цвет при наведении на блоки)
- Движение плавное, с низкой нагрузкой на GPU

Когда фон **выключен** (`data-three-bg="0"` на `<html>`):
- Частицы скрыты
- Страницы отображаются на чистом фоне темы
- Полезно на устройствах без GPU или для экономии энергии

Отключение через **Settings** → **Appearance** → переключатель **Three.js Background**.
Настройка запоминается между сессиями.

| Что | Путь |
|-----|------|
| База данных (теги, кеш, комиксы) | `~/.local/share/MediaVault/MediaVaultDataBase.db` |

## Кредитеншл бэкенд (Credential Store)

API-ключи для Rule34 и Danbooru могут храниться двумя способами:

### GNOME Keyring (KeyringStore)

- Ключи хранятся в системной связке ключей GNOME (libsecret)
- Зашифрованы на уровне ОС
- Требуется пакет `python3-keyring` и работающий GNOME Keyring
- Автоматически выбирается если доступен

### Plain Text (settings.json)

- Ключи хранятся в открытом виде в `~/.config/MediaVault/settings.json`
- Не требует дополнительных зависимостей
- Всегда доступен как fallback

Переключение между бэкендами — в **Settings** → **API Keys** → **Credential Backend**.
Статус активного бэкенда отображается под переключателем.

## Частые вопросы

### Q: Сервер не видит мои файлы
Убедись что в **Settings** → **Appearance** указан правильный путь в поле Media directory. Если папка сменилась — обнови путь и нажми **Scan**.

### Q: Tagfetch не находит теги
1. Проверь API-ключи в **Settings** → **API Keys**
2. Файл может отсутствовать на Rule34 или Danbooru — не всякое изображение есть в их базе
3. MD5-хеш считается на сервере — если файл изменён (даже metadata), хеш не совпадёт

### Q: Превью не грузятся
Нажми **Clean Thumbnail Cache** в **Settings** → **Database** или **Admin Panel** → **Database**, потом обнови страницу. Для видео нужен ffmpeg — должен быть установлен в системе.

### Q: Как перенести всё на другой компьютер
1. Скопируй папку проекта
2. Скопируй `~/.local/share/MediaVault/` (БД с тегами)
3. Скопируй `~/.config/MediaVault/settings.json` (настройки)
4. Запусти сервер

### Q: Не сохраняются теги
Проверь что файл не открыт в другой программе. БД лежит в `~/.local/share/MediaVault/` — проверь права на запись.

### Q: Где настройки и API-ключи?
Настройки на отдельной странице **Settings** (`/settings`) — вкладки Appearance (тема, медиа-папка, эффекты, Three.js фон), API Keys (ключи Rule34/Danbooru, выбор хранилища), Database (инструменты БД), Account (смена логина/пароля). Admin Panel (`/admin`) — отдельная страница для управления пользователями и теми же инструментами БД/API-ключами (для удобства администрирования).

### Q: Я user, а не admin — что мне доступно?
Только просмотр: **MediaVault** (галерея, комиксы, лайтбокс) и **Settings** (только вкладки Appearance и Account).

### Q: Как быстро перейти к Auto-сканированию?
На главной → блок **CONTENT MANAGEMENT** → **Auto** или напрямую `/content-mgmt/tags-auto`.

### Q: Как открыть отдельную страницу просмотра?
В лайтбоксе нажми кнопку ↗ — откроется в новой вкладке.

### Q: Чем отличается Manual от Auto в Tagfetch?
**Manual:** работа с одним файлом, видишь теги до сохранения, можно выбрать Rule34 или Danbooru.
**Auto:** сразу все файлы, SSE-прогресс, результаты в карточках, потом Save All.

### Q: Как переключать режимы в ридере комиксов?
**F** или кнопка в топбаре. Scroll — все страницы вертикально. Lightbox — по одной.

### Q: Не работают визуальные эффекты
Проверь тумблер **Visual Effects** в **Settings** → **Appearance**. Если он выключен — все анимации и размытие отключены.

### Q: Не работает анимированный фон (Three.js)
Проверь, что переключатель **Three.js Background** включён в **Settings** → **Appearance**. Фон работает только на главной странице (`/`) и странице входа (`/login`). Если фон выключен — страницы показываются на чистом фоне темы.

### Q: Где лог ошибок?
В режиме `--debug` лог пишется в `~/.local/share/MediaVault/debug.log`. В консоли — цветной вывод (зелёный=OK, красный=ошибка).
