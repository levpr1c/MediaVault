## base.html — Ядро

### Блоки
- `{% block head %}` — мета-теги, заголовок
- `{% block content %}` — **рендерится ПЕРВЫМ** (до scripts)
- `{% block scripts %}` — **рендерится ВТОРЫМ** (ES модули)
- **Важно:** ошибки верхнего уровня в ES модулях блокируют все остальные скрипты

### Включает
- CSS: shared.css, mediavault.css (порядок важен)
- Three.js фон (home-bg.js) на главной и логине
- Инициализацию лайтбокса
- Sidebar (десктоп) / нижняя панель (мобильная)

### Mobile
- `.desktop-only` элементы скрыты через CSS media query
- Mobile не получает HTML сайдбара/панели поиска/тулбара

## Ключевые шаблоны

### gallery.html (shared/)
- Основной шаблон галереи
- Фильтры: поиск по имени, тегам
- Layout переключатель (masonry/grid/list)
- Включает `popular_tags.html`

### comics-list.html (shared/)
- Список комиксов
- Карточки комиксов (обложка, название, теги)
- Модалка создания/редактирования

### popular_tags.html (shared/)
- Кнопки популярных тегов
- Топ-10/20 тегов по частоте использования
- Один клик — фильтр по тегу

### view.html (shared/)
- Просмотр одного медиафайла
- Информация: имя, размер, разрешение, теги
- Кнопки: скачать, удалить, открыть в лайтбоксе

### admin.html (admin/)
- 5 секций: Users, Database, API Keys, Scan, Mount
- Users: создание/удаление/смена роли/пароля
- Database: экспорт/импорт, очистка, дедупликация, перехеширование
- API Keys: управление ключами Rule34/Danbooru
- Scan: сканирование папок, полный рескан
- Mount: индикатор монтирования хранилища

### tags.html (content-mgmt/)
- Страница управления тегами
- Добавление/удаление тегов к файлам
- Цветовые категории
- Поиск по тегам

## Шаблонные макросы (macros.html)

- `render_tags(tags)` — рендер списка тегов с цветами
- `render_pagination(...)` — пагинация
- `render_media_card(...)` — карточка медиа
- Переиспользуются во многих шаблонах

## ТЕХДОЛГ И УЛУЧШЕНИЯ (для AI-ревью)

### Шаблоны — узкие места

1. **Inline JS в шаблонах**: часть логики живёт в Jinja2, часть в отдельных JS файлах
   - `{% block scripts %}` содержит inline JS, который не линтится и не тестируется
   - Данные передаются через `_i18nData` (глобальная переменная) — не типизировано

2. **Порядок загрузки критичен**: `{% block content %}` рендерится до `{% block scripts %}`
   - Ошибка в любом ES модуле блокирует все остальные скрипты
   - Нет `defer`/`async` — всё синхронно

3. **Мобильная версия через CSS**: `.desktop-only` скрывает элементы, но HTML всё равно грузится
   - Лишний трафик и вес страницы на мобильных

4. **Смешение логики и представления**: шаблоны содержат условные конструкции, которые должны быть в Python или JS
   - `{% if session.role == 'admin' %}` — размазано по всем шаблонам
   - Нет централизованного permission-хелпера

5. **Нет компонентного подхода**: каждый шаблон — законченная страница, нет переиспользуемых компонентов
   - Макросы (`macros.html`) частично решают, но их мало

### Что улучшить

| Задача | Сложность | Эффект |
|--------|-----------|--------|
| Вынести inline JS из шаблонов в отдельные модули | Средняя | Линтинг, тестирование, код-ревью |
| Добавить `defer` к JS модулям | Низкая | Стабильность загрузки |
| Использовать HTMX вместо Jinja2 for partials | Высокая | Меньше JS кода, серверный рендер |
| Централизовать проверки ролей (Python + шаблон) | Низкая | Безопаснее, проще менять политики |
| Компонентный подход: macros.html → Jinja2 components | Средняя | Переиспользование кода |

### Что НЕ трогать
- `base.html` — ядро, менять только с регрессионным тестированием
- Порядок CSS файлов — ломает всю стилизацию
- Финские `{% if session.role == 'admin' %}` — нужны, но стоит вынести в helper
