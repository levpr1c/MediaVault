# Словарь терминов

> Простые объяснения слов и понятий, которые встречаются в MediaVault.

---

## A—Z

### API (Application Programming Interface)
Способ, которым разные программы общаются друг с другом. В MediaVault это HTTP-запросы — браузер (фронтенд) посылает запросы на сервер, сервер отвечает JSON-данными. Например: «покажи список файлов», «сохрани теги».

### AVIF
Современный формат картинок, который сжимает их в 2-3 раза сильнее JPEG без потери качества. MediaVault использует AVIF для миниатюр (превьюшек) — они занимают меньше места в базе данных.

### BLOB (Binary Large Object)
Двоичные данные, которые хранятся прямо в базе данных SQLite. MediaVault хранит миниатюры как BLOB — картинка засовывается в ячейку таблицы. Удобно: всё в одном файле, не надо таскать папки с превью.

### Booru-сайты (booru)
Сайты вроде Danbooru, Rule34, Gelbooru, где картинки размечены тегами. MediaVault умеет искать теги на Rule34.xxx и Danbooru.donmai.us по MD5-хешу файла.

### CSS (Cascading Style Sheets)
Язык, который отвечает за внешний вид страниц: цвета, шрифты, отступы, анимации. В MediaVault — 6 CSS-файлов.

### Danbooru
Огромный сайт с аниме-артом, где каждая картинка размечена тегами по категориям (artist, character, copyright, general, meta). MediaVault может автоматически находить теги на Danbooru по MD5 файла.

### E-Hentai
Крупнейшая галерея изображений и манги для взрослых (дочерний сайт HentaiVerse). MediaVault поддерживает поиск и загрузку контента через gallery-dl. API-эндпоинт: `https://api.e-hentai.org/api.php`.

### Flask
Лёгкий веб-фреймворк на Python. Это «движок» MediaVault — он принимает запросы от браузера, работает с базой данных, отдаёт HTML-страницы и JSON-ответы.

### FFmpeg
Программа для работы с видео и аудио. MediaVault использует FFmpeg, чтобы:
- Сделать превью-кадр из видео
- Определить, есть ли в файле звук (для авто-тега `sound`)

### i18n (internationalization)
Система перевода интерфейса на разные языки. В MediaVault — английский и русский. Переключается без перезагрузки страницы. Название происходит от количества букв между i и n в слове "internationalization" (18).

### Jinja2
Язык шаблонов для Python. MediaVault использует его, чтобы вставлять данные (настройки, теги, список файлов) прямо в HTML на сервере. Например, `{{ _('save') }}` вставит слово «Сохранить» на русском или «Save» на английском.

### JSON (JavaScript Object Notation)
Простой формат для передачи данных. Выглядит так: `{"username": "admin", "role": "admin"}`. Все API MediaVault отвечают в JSON.

### Lightbox
Режим просмотра, когда картинка открывается на весь экран с тёмным фоном, навигацией (вперёд/назад) и зумом. В MediaVault два режима лайтбокса — отдельные файлы и комиксы.

### Masonry
Раскладка галереи, когда картинки разной высоты располагаются как кирпичная кладка — без пустых мест. MediaVault использует CSS columns для masonry.

### MD5 (хеш)
Короткая «подпись» файла (32 символа). Если у двух файлов одинаковый MD5 — это 100% один и тот же файл. Booru-сайты хранят MD5 для каждой картинки, по нему MediaVault ищет теги.

### Pillow (PIL)
Библиотека для работы с картинками на Python. MediaVault использует её для:
- Определения размеров изображения (ширина × высота)
- Создания превью (уменьшения картинок)
- Определения типа файла

### Rule34
Booru-сайт, специализирующийся на взрослом контенте, с большим архивом тегов. MediaVault умеет искать теги на Rule34 по MD5 файла.

### SSE (Server-Sent Events)
Способ, которым сервер может отправлять данные браузеру в реальном времени. Используется в Auto-режиме Tagfetch, чтобы показывать прогресс сканирования: «обработано 5 из 100 файлов».

### SPA (Single Page Application)
Страница, которая не перезагружается целиком при каждом действии. В MediaVault так работают: Content Management, Admin Panel, Settings. Все данные загружаются через API без перезагрузки HTML.

### SQLite
Лёгкая встраиваемая база данных. Хранится в одном файле (`~/.local/share/MediaVault/MediaVaultDataBase.db`). Не требует установки сервера.

### Three.js
JavaScript библиотека для 3D-графики. MediaVault использует её для анимированного фона на домашней странице и странице логина. Шейдерный фон работает на GPU. Версия 0.160.0, файл 53044 строк, расположен в `/static/lib/three.module.js`.

### Content Search
Страница унифицированного поиска `/content-search`, позволяющая искать изображения и мангу одновременно на Rule34, Danbooru, NHentai и E-Hentai. Результаты показываются в карточках с превью, а в лайтбоксе — с цветными категориями тегов и кнопкой скачивания (с указанием сайта-источника).

### AI Filter
Чекбокс в Content Search, исключающий AI-сгенерированные изображения из результатов Rule34 путём добавления тегов `-ai_generated -ai -ai_assisted` к запросу.

### Mount Indicator (индикатор монтирования)
Мигающая точка в интерфейсе (Settings и Admin Panel), показывающая статус подключения storage-накопителя: зелёная — смонтирован, красная — не смонтирован или пуст.

### downloadLabelFn
Опция конструктора Lightbox, позволяющая задать динамический текст для кнопки скачивания. В Content Search показывает название сайта-источника (например, «Скачать с Danbooru»).

### Manga Download (NHentai)
Функция загрузки всех страниц NHentai-галереи в папку `Downloads/nhentai/{gid}/` с полным индексированием в БД. Запускается из лайтбокса Content Search.

### tags_by_category
Словарь, возвращаемый API content-search, содержащий теги Danbooru, сгруппированные по категориям: `{artist: [...], character: [...], copyright: [...], general: [...], meta: [...]}`. Используется для цветовой маркировки тегов в лайтбоксе.

### importmap
HTML-тег `<script type="importmap">`, который указывает браузеру где искать ES-модули. В MediaVault используется для указания пути к Three.js: `"three": "/static/lib/three.module.js"`.

## А—Я

### Авто-теги (auto-tags)
Теги, которые MediaVault присваивает сам при сканировании, без обращения к booru-сайтам:
- `photo` — обычное изображение
- `video` — видеофайл
- `animated` — гифка или анимированное изображение  
- `gif` — файл в формате GIF
- `sound` — в файле есть аудиодорожка
- `16:9`, `4:3` и т.д. — соотношение сторон

### Админ-панель (Admin Panel)
Страница `/admin`, где администратор может управлять пользователями, API-ключами, базой данных и настройками.

### Декоратор
Пометка над функцией в Python, которая добавляет ей дополнительное поведение. В MediaVault:
- `@admin_required` — только для админов (33 штуки)
- `@auth_required` — для любого авторизованного пользователя (2 штуки)
- `@api_error_handler` — автоматически ловит ошибки и возвращает JSON (53 штуки)

### Категории тегов (tag categories)
Группы тегов с цветами. MediaVault поддерживает Danbooru-категории:
- **Artist** (голубой) — художник
- **Character** (оранжевый) — персонаж
- **Copyright** (фиолетовый) — серия/франшиза
- **General** (зелёный) — общие теги
- **Meta** (синий) — служебные теги
- **User** — свои категории

### Комикс (comic)
Подборка файлов (обычно картинок), собранных в определённом порядке для последовательного чтения. MediaVault поддерживает два режима чтения: Scroll (лента) и Lightbox (постранично).

### Медиа-директория (media_dir)
Путь к папке на компьютере, где лежат медиафайлы. Указывается в настройках. MediaVault сканирует эту папку и добавляет файлы в базу.

### Миниатюра (thumbnail, превью)
Уменьшенная копия картинки или видео для быстрого показа в галерее. Хранится в базе данных в формате AVIF. Если миниатюры нет — MediaVault создаст её автоматически при первом просмотре.

### Пользователь (user)
Учётная запись с ролью `user`. Может только просматривать галерею и комиксы. Не может редактировать теги, загружать файлы, менять настройки.

### Сканирование (scan)
Процесс, при котором MediaVault обходит медиа-папку, находит новые файлы и добавляет их в базу с авто-тегами.

### Тег (tag)
Ключевое слово, которое описывает содержимое файла. Например: `cat` (кошка), `sunset` (закат), `1girl` (одна девушка). Теги можно искать, фильтровать и группировать по категориям.

### Хранилище ключей (credential store)
Место, где хранятся API-ключи для Rule34 и Danbooru. Варианты:
- **KeyringStore** — в системной связке GNOME Keyring (безопасно)
- **Plain Text** — в файле настроек `settings.json` (менее безопасно)

### Хеш (hash)
Смотри **MD5**.
