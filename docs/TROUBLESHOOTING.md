# Решение проблем (Troubleshooting)

> Собрание типичных проблем и способов их решить. Если твоей проблемы здесь нет — проверь [FAQ](FAQ.md) или заведи issue на GitHub.

## 🚫 Сервер не запускается

### Port 5050 already in use
```
Address already in use
```
Порт занят другой программой. Что делать:
```bash
# Узнать, кто занял порт
sudo lsof -i :5050

# Запустить на другом порту
venv/bin/python src/web_app.py --port 8080

# Или убить процесс
kill $(lsof -t -i :5050)
```

### ImportError: No module named 'flask'
```
ModuleNotFoundError: No module named 'flask'
```
Не установлены зависимости:
```bash
venv/bin/pip install flask requests Pillow
```

### FFmpeg not found
При запуске или открытии видео:
```
FFmpeg не установлен
```
Установи FFmpeg:
```bash
# Ubuntu/Debian
sudo apt install ffmpeg

# Arch
sudo pacman -S ffmpeg

# Fedora
sudo dnf install ffmpeg

# macOS
brew install ffmpeg
```

## 🔐 Авторизация

### Не могу войти
Проверь:
1. Сервер запущен
2. Открываешь `http://localhost:5050` (не `https://`)
3. Если вводил имя пользователя — проверь раскладку клавиатуры

### Забыл пароль администратора
Сброс через SQLite:
```bash
sqlite3 ~/.local/share/MediaVault/MediaVaultDataBase.db \
  "UPDATE users SET password='' WHERE role='admin' LIMIT 1"
```
После этого зайди без пароля и установи новый.

### Нет кнопок редактирования
Ты вошёл как `user`, а не `admin`. Пользователь с ролью `user` может только просматривать галерею и комиксы. Войди как администратор.

### Приложение требует пароль, но я его не ставил
Первый запуск — зарегистрируй администратора. Если база уже есть — смотри пункт «Забыл пароль администратора».

## 📁 Медиа и галерея

### Нет файлов в галерее
Пошагово:
1. **Settings → Appearance** — проверь, что указан правильный путь к медиа-папке
2. Нажми **Scan Folder** — файлы должны появиться
3. Если не появились — проверь, что папка существует и в ней есть файлы
4. Убедись, что расширения файлов поддерживаются (jpg, png, gif, webp, mp4, webm и т.д.)

### Файлы есть, но в галерее пусто
```bash
curl -X POST http://localhost:5050/api/scan_folder
```
Должен вернуть `{"scanning": true}`.

### Миниатюры (превью) не показываются
1. Проверь, что файлы существуют на диске
2. Перегенерируй миниатюры:
   - **Settings → Database → Regenerate All Thumbnails** (или Generate Missing)
   - Или через API: `curl -X POST http://localhost:5050/api/regenerate_thumbnails`
3. Для видео — проверь, что установлен FFmpeg

### Некоторые миниатюры чёрные/пустые
Скорее всего, файл повреждён или имеет неподдерживаемый формат. Проверь сам файл — открывается ли он обычным просмотрщиком?

### Видео не проигрывается
Браузеры не поддерживают все форматы видео. Убедись, что формат (mp4, webm) поддерживается браузером. Если видео без звука — может не работать в некоторых браузерах.

## 🏷️ Теги

### Теги не находятся на Rule34/Danbooru
1. Проверь API-ключи: **Settings → API Keys**
2. Нажми **Save** после ввода ключей
3. Попробуй вручную: **Tagfetch → Manual** — выбери файл, нажми «Fetch Tags»
4. Файла может не быть на booru-сайте (особенно если это твоя личная фотография)
5. Для Rule34: ключи можно получить в профиле на rule34.xxx после регистрации

### Ошибка «Invalid API key» на Rule34
Ключи неверные или истекли. Проверь на Rule34.xxx — возможно, нужно обновить.

### Авто-теги не ставятся
Авто-теги (photo, video, animated, sound, соотношение сторон) ставятся ТОЛЬКО при сканировании (Scan Folder или Auto Scan). Если ты добавил файл в папку вручную — запусти сканирование.

### Как удалить все теги?
**Settings → Database → Clean All** — полная очистка.
Или **Clean Tag Cache** — очистка только кэша результатов поиска (теги файлов сохранятся).

## 🎴 Комиксы

### Кнопка создания комикса не появляется
Ты должен быть авторизован как `admin`. Кнопка есть:
- В просмотрщике (`/mediavault/view` → «Create comic»)
- В Content Management → COMICS EDITOR

### Не могу перетащить страницы в редакторе
Убедись, что не зажата случайно клавиша Ctrl или Shift. Попробуй обновить страницу.

### Комикс не сохраняется
Проверь:
1. Есть ли у комикса название
2. Есть ли хотя бы одна страница
3. Нажал ли «Save» (кнопка в правом верхнем углу редактора)

## ⚙️ Настройки

### «Credential Backend: KeyringStore not available»
Системная связка GNOME Keyring недоступна. MediaVault автоматически переключится на хранение в `settings.json` (обычный текст). Это нормально, все функции будут работать. Можно переключить вручную в Settings → Credential Backend.

### Настройки не сохраняются
Проверь права на запись:
```bash
ls -la ~/.config/MediaVault/
ls -la ~/.config/MediaVault/settings.json
```
Файл должен быть доступен для записи.

### База данных не экспортируется
```bash
# Вручную скопировать
cp ~/.local/share/MediaVault/MediaVaultDataBase.db ./backup.db
```

## 🖥️ Интерфейс

### Неправильно отображаются элементы
1. Очисти кэш браузера (Ctrl+F5)
2. Попробуй другой браузер
3. Проверь, включены ли визуальные эффекты (Settings → Visual Effects)

### Язык не переключается
Кнопка с глобусом в правом верхнем углу. Если текст не обновляется — нажми Ctrl+F5.

### Тёмная тема не применяется
Проверь: Settings → Appearance → кнопка темы (солнце/луна). Если не помогает — очисти localStorage:
```javascript
// Выполни в консоли браузера (F12)
localStorage.removeItem('mediavault_theme');
```

### Мобильная версия выглядит странно
MediaVault адаптируется под экран, но это не мобильное приложение. На телефоне может быть неудобно. Рекомендуется экран шириной от 768px.

### Не работает поиск в Content Management (Content Search)
Если кнопка поиска не реагирует или страница загружается некорректно — очисти кэш браузера (Ctrl+F5). Content Search использует ES-модули, которые агрессивно кэшируются браузером. После обновления JavaScript может потребоваться полная очистка кэша.

### Content Search не загружается (TypeError в консоли: «Cannot read properties of null»)
Ошибка `TypeError: Cannot read properties of null (reading 'querySelector')` возникает когда `Lightbox.close()` вызывается до того, как лайтбокс был открыт (элементы DOM ещё не созданы). Исправлено в `static/shared/lightbox.js:279` — добавлена проверка на null. Если проблема повторяется — очисти кэш браузера (Ctrl+F5).

## 🐍 Ошибки Python

### TypeError: not all arguments converted during string formatting
Проблема с форматированием строк в старом коде. Сообщи разработчику. Временно может помочь перезапуск.

### sqlite3.OperationalError: no such table
База данных повреждена или не создана. Останови сервер, удали `MediaVaultDataBase.db` и запусти снова — таблицы создадутся автоматически.

### Server error 500
Полная ошибка обычно видна в консоли (если запуск с `--debug`). Посмотри последние строки — там будет Python traceback.

## 💾 Резервное копирование

### Как сделать бекап?
```bash
# База данных
cp ~/.local/share/MediaVault/MediaVaultDataBase.db ~/backup-$(date +%Y%m%d).db

# Настройки
cp ~/.config/MediaVault/settings.json ~/backup-settings-$(date +%Y%m%d).json
```

### Как восстановить из бекапа?
1. Останови сервер
2. Скопируй файлы обратно:
   ```bash
   cp ~/backup-20260101.db ~/.local/share/MediaVault/MediaVaultDataBase.db
   cp ~/backup-settings-20260101.json ~/.config/MediaVault/settings.json
   ```
3. Запусти сервер заново
